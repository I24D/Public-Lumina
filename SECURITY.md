# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `1.0.10` | Yes |
| Older public builds | No |

## Reporting a Vulnerability

Do not disclose security issues publicly before the maintainers have reviewed them.

If GitHub private vulnerability reporting is enabled for this repository, use that channel. Otherwise, open a minimal public issue titled `Security contact request` without including exploit details, credentials, tokens, private logs, or sensitive user data.

## Secrets Policy

This repository must never contain:

- API keys or tokens.
- `.env` files.
- Private service credentials.
- User data.
- Internal-only source or deployment secrets.
